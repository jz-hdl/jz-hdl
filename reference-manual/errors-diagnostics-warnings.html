Errors and Diagnostics ‚Äã

Severity Levels ‚Äã

Every diagnostic emitted by the JZ-HDL compiler has one of three severity levels:

SeverityMeaningEffect
ERRORThe design violates a language rule and cannot be compiled.Compilation stops. No output is produced.
WARNINGThe design is legal but likely contains an unintended issue (unused declarations, missing DEFAULT clauses, etc.).Compilation continues. The design may work, but you should investigate.
INFOAdvisory information about the design (e.g., runtime division-by-zero risk, blackbox overrides that cannot be validated).Compilation continues. Informational only ‚Äî no action required unless the note is relevant to your intent.

Controlling Diagnostic Output ‚Äã

CLI FlagEffect
--infoShow INFO-level diagnostics in addition to errors and warnings. Without this flag, only errors and warnings are displayed.
--warn-as-errorPromote all warnings to errors. Useful for CI pipelines where you want zero tolerance for potential issues.
--Wno-group=NAMESuppress all diagnostics in the named group. For example, --Wno-group=GENERAL_WARNINGS silences unused register warnings, dead code warnings, etc.
--colorForce colored output. Errors appear in red, warnings in yellow, and info in blue.

Listing All Rules ‚Äã

Use --lint-rules to print every diagnostic rule ID, its severity, and a short description:

bash
jz-hdl design.jz --lint-rules

This is useful for discovering the exact rule ID to reference when suppressing diagnostics with --Wno-group, or when searching this page for details about a specific diagnostic.

Diagnostic Reference ‚Äã

The sections below list all diagnostic rules organized by category. Each entry shows the rule ID, severity, typical cause, and how to fix it.

Parse & Lexical ‚Äã

PARSE.COMMENT_IN_TOKEN ‚Äî Comment appears inside a token ‚Äã

  • Severity: ERROR
  • Cause: /* or // started inside an identifier/number/operator token.
  • Fix: Move comment boundaries so they appear between tokens or on separate lines.

PARSE.COMMENT_NESTED_BLOCK ‚Äî Nested block comment detected ‚Äã

  • Severity: ERROR
  • Cause: /* ... /* ... */ ... */ ‚Äî nested block comments are not supported.
  • Fix: Remove inner /* or close the outer block before starting another; use line comments instead.

PARSE.DIRECTIVE_INVALID_CONTEXT ‚Äî Directive used in invalid location ‚Äã

  • Severity: ERROR
  • Cause: Structural directives (@project, @module, @import, @new, @blackbox, @endproj, @endmod, etc.) placed where not allowed.
  • Fix: Place directives only in permitted locations per the spec (e.g., @import only immediately inside @project).

LEXICAL.ID_SYNTAX_INVALID ‚Äî Identifier invalid ‚Äã

  • Severity: ERROR
  • Cause: Identifier violates regex (bad characters, starts with digit) or length > 255.
  • Fix: Rename to valid identifier per rules: [A‚ÄëZa‚Äëz_][A‚ÄëZa‚Äëz0‚Äë9_]{0,254} and not a single underscore.

LEXICAL.ID_SINGLE_UNDERSCORE ‚Äî Illegal use of single underscore ‚Äã

  • Severity: ERROR
  • Cause: _ used as regular identifier outside no‚Äëconnect context (allowed only in @top/@new port bindings).
  • Fix: Use a named identifier or _ only in permitted instantiation/top-level no‚Äëconnect positions.

PARSE.KEYWORD_AS_IDENTIFIER ‚Äî Reserved keyword used as identifier ‚Äã

  • Severity: ERROR
  • Cause: Using a reserved uppercase keyword as an identifier.
  • Fix: Rename; reserved keywords (IF, ELSE, CONST, IN, OUT, etc.) cannot be used as identifiers.

PARSE.IF_COND_MISSING_PARENS ‚Äî IF/ELIF condition missing parentheses ‚Äã

  • Severity: ERROR
  • Cause: IF cond { instead of IF (cond) {.
  • Fix: Parenthesize the condition.

PARSE.LIT_DECIMAL_HAS_XZ ‚Äî Decimal literal contains x or z ‚Äã

  • Severity: ERROR
  • Cause: Decimal literals do not support x/z digits (e.g., 8'd10x).
  • Fix: Use hex or binary base for x/z values.

PARSE.LIT_INVALID_DIGIT_FOR_BASE ‚Äî Invalid digit for literal base ‚Äã

  • Severity: ERROR
  • Cause: Literal contains a digit not valid for its base (e.g., 8'b102).
  • Fix: Use valid digits for the base.

PARSE.INSTANCE_UNDEFINED_MODULE ‚Äî Instantiation references non-existent module ‚Äã

  • Severity: ERROR
  • Fix: Ensure the module is defined or imported before use.

Literals, Types & Widths ‚Äã

LITERALS_AND_TYPES.LIT_UNSIZED ‚Äî Unsized literal used ‚Äã

  • Severity: ERROR
  • Cause: Literal missing width (e.g., 'hFF).
  • Fix: Provide explicit width: 8'hFF.

LITERALS_AND_TYPES.LIT_OVERFLOW ‚Äî Literal intrinsic width > declared width ‚Äã

  • Severity: ERROR
  • Cause: Value requires more bits than declared.
  • Fix: Increase literal width or shorten the value so intrinsic width ‚â§ declared width.

LITERALS_AND_TYPES.LIT_UNDERSCORE_AT_EDGES ‚Äî Underscore at start/end of literal ‚Äã

  • Severity: ERROR
  • Cause: Literal has leading or trailing _.
  • Fix: Remove underscores at edges.

LITERALS_AND_TYPES.LIT_UNDEFINED_CONST_WIDTH ‚Äî Undefined CONST used as width ‚Äã

  • Severity: ERROR
  • Cause: Using a module CONST name in a literal width that does not exist.
  • Fix: Declare the CONST or use a numeric width; CONST scope is module-local.

WIDTHS_AND_SLICING.WIDTH_NONPOSITIVE_OR_NONINT ‚Äî Invalid declared width ‚Äã

  • Severity: ERROR
  • Cause: Declared width <= 0 or not integer.
  • Fix: Use positive integer or valid CONST resolving to positive integer.

WIDTHS_AND_SLICING.SLICE_MSB_LESS_THAN_LSB ‚Äî Slice MSB < LSB ‚Äã

  • Severity: ERROR
  • Cause: signal[H:L] where H < L.
  • Fix: Reverse indices or correct indices so MSB ‚â• LSB.

WIDTHS_AND_SLICING.SLICE_INDEX_OUT_OF_RANGE ‚Äî Slice index out of bounds ‚Äã

  • Severity: ERROR
  • Cause: Index < 0 or >= signal width.
  • Fix: Use valid indices within [0, width‚àí1].

OPERATORS_AND_EXPRESSIONS.TYPE_BINOP_WIDTH_MISMATCH ‚Äî Binary op operands differ widths ‚Äã

  • Severity: ERROR
  • Cause: Operators requiring equal widths (e.g., +, &, ==) receive different widths.
  • Fix: Make operand widths equal via explicit extension (=z/=s context) or resize using concatenation or intrinsics.

Operators & Expressions ‚Äã

OPERATORS_AND_EXPRESSIONS.UNARY_ARITH_MISSING_PARENS ‚Äî Unary arithmetic missing parentheses ‚Äã

  • Severity: ERROR
  • Cause: Used -flag instead of (-flag).
  • Fix: Parenthesize unary arithmetic: (-flag).

OPERATORS_AND_EXPRESSIONS.LOGICAL_WIDTH_NOT_1 ‚Äî &&/||/! used on multi-bit ‚Äã

  • Severity: ERROR
  • Cause: Logical operators require width‚Äë1 operands.
  • Fix: Reduce to a single-bit condition (e.g., (a != 0)), or use bitwise operators.

OPERATORS_AND_EXPRESSIONS.TERNARY_COND_WIDTH_NOT_1 ‚Äî Ternary condition not width‚Äë1 ‚Äã

  • Severity: ERROR
  • Fix: Ensure the condition expression is width‚Äë1.

OPERATORS_AND_EXPRESSIONS.TERNARY_BRANCH_WIDTH_MISMATCH ‚Äî Branches widths differ ‚Äã

  • Severity: ERROR
  • Cause: true/false expressions have different widths.
  • Fix: Make branches equal width by extending or slicing.

OPERATORS_AND_EXPRESSIONS.CONCAT_EMPTY ‚Äî Empty concatenation {} used ‚Äã

  • Severity: ERROR
  • Fix: Remove empty concat or supply elements.

OPERATORS_AND_EXPRESSIONS.DIV_CONST_ZERO ‚Äî Division by constant zero ‚Äã

  • Severity: ERROR
  • Cause: A compile-time divisor literal = 0.
  • Fix: Remove or guard division; use conditional checks.

OPERATORS_AND_EXPRESSIONS.DIV_UNGUARDED_RUNTIME_ZERO ‚Äî Divisor may be zero at runtime ‚Äã

  • Severity: WARNING
  • Cause: A division or modulus with a non-constant divisor is not enclosed in an IF (divisor != 0) guard.
  • Fix: Add runtime guards: IF (divisor != 0) ... ELSE ....

OPERATORS_AND_EXPRESSIONS.SPECIAL_DRIVER_IN_EXPRESSION ‚Äî GND/VCC in expression ‚Äã

  • Severity: ERROR
  • Cause: GND/VCC may not appear in arithmetic/logical expressions.
  • Fix: Use sized literals (1'b0, 1'b1) instead.

OPERATORS_AND_EXPRESSIONS.SPECIAL_DRIVER_IN_CONCAT ‚Äî GND/VCC in concatenation ‚Äã

  • Severity: ERROR
  • Fix: Use sized literals instead of GND/VCC in concatenations.

OPERATORS_AND_EXPRESSIONS.SPECIAL_DRIVER_SLICED ‚Äî GND/VCC sliced or indexed ‚Äã

  • Severity: ERROR
  • Fix: GND/VCC are whole-signal drivers; use sized literals if you need specific bits.

OPERATORS_AND_EXPRESSIONS.SPECIAL_DRIVER_IN_INDEX ‚Äî GND/VCC in index expression ‚Äã

  • Severity: ERROR
  • Fix: Use integer literals for indices.

OPERATORS_AND_EXPRESSIONS.SBIT_SET_WIDTH_NOT_1 ‚Äî sbit() set argument not width-1 ‚Äã

  • Severity: ERROR
  • Fix: The third argument to sbit() must be a 1-bit expression.

OPERATORS_AND_EXPRESSIONS.GBIT_INDEX_OUT_OF_RANGE ‚Äî gbit() index out of range ‚Äã

  • Severity: ERROR
  • Fix: Ensure index is within [0, width(source)-1].

OPERATORS_AND_EXPRESSIONS.GSLICE_INDEX_OUT_OF_RANGE ‚Äî gslice() index out of range ‚Äã

  • Severity: ERROR
  • Fix: Ensure index + width <= width(source).

OPERATORS_AND_EXPRESSIONS.GSLICE_WIDTH_INVALID ‚Äî gslice() width invalid ‚Äã

  • Severity: ERROR
  • Fix: Width parameter must be a positive integer constant.

OPERATORS_AND_EXPRESSIONS.SSLICE_INDEX_OUT_OF_RANGE ‚Äî sslice() index out of range ‚Äã

  • Severity: ERROR
  • Fix: Ensure index + width <= width(source).

OPERATORS_AND_EXPRESSIONS.SSLICE_WIDTH_INVALID ‚Äî sslice() width invalid ‚Äã

  • Severity: ERROR
  • Fix: Width parameter must be a positive integer constant.

OPERATORS_AND_EXPRESSIONS.SSLICE_VALUE_WIDTH_MISMATCH ‚Äî sslice() value width mismatch ‚Äã

  • Severity: ERROR
  • Fix: Value argument width must match the width parameter.

OPERATORS_AND_EXPRESSIONS.OH2B_INPUT_TOO_NARROW ‚Äî oh2b() source too narrow ‚Äã

  • Severity: ERROR
  • Fix: oh2b() source must be at least 2 bits wide.

OPERATORS_AND_EXPRESSIONS.B2OH_WIDTH_INVALID ‚Äî b2oh() width invalid ‚Äã

  • Severity: ERROR
  • Fix: Width must be a positive constant >= 2.

OPERATORS_AND_EXPRESSIONS.PRIENC_INPUT_TOO_NARROW ‚Äî prienc() source too narrow ‚Äã

  • Severity: ERROR
  • Fix: prienc() source must be at least 2 bits wide.

OPERATORS_AND_EXPRESSIONS.BSWAP_WIDTH_NOT_BYTE_ALIGNED ‚Äî bswap() width not byte-aligned ‚Äã

  • Severity: ERROR
  • Fix: Source width must be a multiple of 8.

Identifiers & Scope ‚Äã

IDENTIFIERS_AND_SCOPE.ID_DUP_IN_MODULE ‚Äî Duplicate identifier in module ‚Äã

  • Severity: ERROR
  • Cause: Two declarations with same name (ports, wires, registers, consts, instances).
  • Fix: Rename one declaration; ensure uniqueness in module scope.

IDENTIFIERS_AND_SCOPE.MODULE_NAME_DUP_IN_PROJECT ‚Äî Module name not unique project-wide ‚Äã

  • Severity: ERROR
  • Fix: Rename module or resolve import duplication.

IDENTIFIERS_AND_SCOPE.INSTANCE_NAME_DUP_IN_MODULE ‚Äî Duplicate instance name ‚Äã

  • Severity: ERROR
  • Fix: Use unique instance names.

IDENTIFIERS_AND_SCOPE.INSTANCE_NAME_CONFLICT ‚Äî Instance name conflicts other identifier ‚Äã

  • Severity: ERROR
  • Fix: Rename instance so it doesn't collide with port/wire/register/CONST.

IDENTIFIERS_AND_SCOPE.UNDECLARED_IDENTIFIER ‚Äî Undeclared name used ‚Äã

  • Severity: ERROR
  • Fix: Declare the identifier or correct the name.

IDENTIFIERS_AND_SCOPE.AMBIGUOUS_REFERENCE ‚Äî Ambiguous reference ‚Äã

  • Severity: ERROR
  • Cause: Name could refer to multiple things without instance/qualification.
  • Fix: Qualify with instance.port or rename conflicting identifiers.

CONST / CONFIG / GLOBAL ‚Äã

CONST_RULES.CONST_NEGATIVE_OR_NONINT ‚Äî CONST invalid value ‚Äã

  • Severity: ERROR
  • Fix: Initialize CONST with nonnegative integer.

CONFIG_BLOCK.CONFIG_MULTIPLE_BLOCKS ‚Äî Multiple CONFIG blocks ‚Äã

  • Severity: ERROR
  • Fix: Merge into single CONFIG block.

CONFIG_BLOCK.CONFIG_FORWARD_REF ‚Äî CONFIG forward reference ‚Äã

  • Severity: ERROR
  • Cause: CONFIG entry references a later CONFIG entry.
  • Fix: Reorder entries or use computed constants appropriately.

CONFIG_BLOCK.CONFIG_USED_WHERE_FORBIDDEN / CONST_USED_WHERE_FORBIDDEN ‚Äã

  • Severity: ERROR
  • Cause: CONFIG/CONST used at runtime (in ASYNCHRONOUS/SYNCHRONOUS expressions).
  • Fix: Use @global sized literals for runtime values; CONFIG/CONST only for compile-time contexts.

CONFIG_BLOCK.CONST_STRING_IN_NUMERIC_CONTEXT ‚Äî String CONST/CONFIG in numeric context ‚Äã

  • Severity: ERROR
  • Cause: A string-valued CONST or CONFIG entry is used where a numeric expression is expected (e.g., in a width bracket [STR_CONST] or MEM depth).
  • Fix: Use a numeric CONST/CONFIG for integer contexts. String values are only valid in @file() path arguments.

CONFIG_BLOCK.CONST_NUMERIC_IN_STRING_CONTEXT ‚Äî Numeric CONST/CONFIG in string context ‚Äã

  • Severity: ERROR
  • Cause: A numeric CONST or CONFIG entry is used where a string is expected (e.g., @file(NUM_CONST)).
  • Fix: Use a string CONST/CONFIG for @file() paths (e.g., INIT_FILE = "data.bin";).

GLOBAL_BLOCK.GLOBAL_INVALID_EXPR_TYPE ‚Äî Global constant must be sized literal ‚Äã

  • Severity: ERROR
  • Fix: Define global constants as sized literals (e.g., 8'hFF).

Ports, Wires & Registers ‚Äã

PORT_WIRE_REGISTER_DECLS.PORT_MISSING_WIDTH ‚Äî Port declared without width ‚Äã

  • Severity: ERROR
  • Fix: Add mandatory [N] width.

PORT_WIRE_REGISTER_DECLS.PORT_DIRECTION_MISMATCH_IN ‚Äî Assigning to IN port ‚Äã

  • Severity: ERROR
  • Cause: Writing to an IN port outside allowed INOUT tri‚Äëstate patterns.
  • Fix: Only read IN ports; if bidirectional behavior is needed use INOUT and tri‚Äëstate logic.

PORT_WIRE_REGISTER_DECLS.PORT_DIRECTION_MISMATCH_OUT ‚Äî Reading from OUT port ‚Äã

  • Severity: ERROR
  • Cause: Reading an OUT port as if it were a driver.
  • Fix: Do not read module OUT ports inside the same module; use internal regs/wires for observation.

PORT_WIRE_REGISTER_DECLS.PORT_TRISTATE_MISMATCH ‚Äî z assigned to non-INOUT ‚Äã

  • Severity: ERROR
  • Fix: Only assign z to INOUT ports.

PORT_WIRE_REGISTER_DECLS.WIRE_MULTI_DIMENSIONAL / REG_MULTI_DIMENSIONAL ‚Äî Multi-dimensional illegal ‚Äã

  • Severity: ERROR
  • Fix: Use MEM for arrays; declare WIRE/REGISTER as single-dimensional scalars.

PORT_WIRE_REGISTER_DECLS.REG_MISSING_INIT_LITERAL ‚Äî Register missing reset literal ‚Äã

  • Severity: ERROR
  • Fix: Provide a sized literal reset value (no x bits).

PORT_WIRE_REGISTER_DECLS.REG_INIT_CONTAINS_X ‚Äî Register reset contains x ‚Äã

  • Severity: ERROR
  • Fix: Use known 0/1 bits for reset; x not permitted.

PORT_WIRE_REGISTER_DECLS.REG_INIT_CONTAINS_Z ‚Äî Register reset contains z ‚Äã

  • Severity: ERROR
  • Fix: Use known 0/1 bits for reset; z not permitted.

PORT_WIRE_REGISTER_DECLS.REG_INIT_WIDTH_MISMATCH ‚Äî Register init width mismatch ‚Äã

  • Severity: ERROR
  • Fix: Init literal width must match declared register width.

PORT_WIRE_REGISTER_DECLS.WRITE_WIRE_IN_SYNC ‚Äî Writing wire in SYNCHRONOUS ‚Äã

  • Severity: ERROR
  • Fix: Writes to wires must occur in ASYNCHRONOUS; registers are updated in SYNCHRONOUS blocks.

PORT_WIRE_REGISTER_DECLS.ASSIGN_TO_NON_REGISTER_IN_SYNC ‚Äî LHS not a register in sync assignment ‚Äã

  • Severity: ERROR
  • Fix: Use a REGISTER on LHS for synchronous assignments.

PORT_WIRE_REGISTER_DECLS.MODULE_MISSING_PORT / MODULE_PORT_IN_ONLY ‚Äã

  • Severity: ERROR / WARN
  • Fix: Ensure module declares PORT block and includes outputs if expected; a module with only IN ports may be dead code.

Modules & Instantiation ‚Äã

MODULE_AND_INSTANTIATION.INSTANCE_MISSING_PORT ‚Äî Not all child ports listed in @new ‚Äã

  • Severity: ERROR
  • Fix: Include all child ports in the @new instantiation block.

MODULE_AND_INSTANTIATION.INSTANCE_PORT_WIDTH_MISMATCH ‚Äî Port width mismatch ‚Äã

  • Severity: ERROR
  • Fix: Ensure evaluated widths (after OVERRIDE) match between parent binding and child port.

MODULE_AND_INSTANTIATION.INSTANCE_PORT_DIRECTION_MISMATCH ‚Äî Direction mismatch ‚Äã

  • Severity: ERROR
  • Fix: Connect IN ports to drivers, OUT ports to sinks or parent signals with correct roles; respect pin category.

MODULE_AND_INSTANTIATION.INSTANCE_OVERRIDE_CONST_UNDEFINED ‚Äî Override targets unknown ‚Äã

  • Severity: ERROR
  • Fix: Only override CONST names that exist in the child module.

MODULE_AND_INSTANTIATION.INSTANCE_PARENT_SIGNAL_WIDTH_MISMATCH ‚Äî Parent signal width wrong ‚Äã

  • Severity: ERROR
  • Fix: Ensure parent signal has the width declared in instantiation clause (check parent's CONST resolution).

MUX Rules ‚Äã

MUX_RULES.MUX_ASSIGN_LHS ‚Äî Assigning to MUX is forbidden ‚Äã

  • Severity: ERROR
  • Fix: Treat MUX only as read-only source; create a wire/register to hold selected data if you must drive it.

MUX_RULES.MUX_AGG_SOURCE_WIDTH_MISMATCH ‚Äî Aggregation widths differ ‚Äã

  • Severity: ERROR
  • Fix: Make all aggregated sources identical width before declaring the MUX.

MUX_RULES.MUX_SLICE_WIDTH_NOT_DIVISOR ‚Äî Auto-slicing invalid partition ‚Äã

  • Severity: ERROR
  • Fix: Choose element_width that exactly divides wide_source width.

MUX_RULES.MUX_SELECTOR_OUT_OF_RANGE_CONST ‚Äî Selector statically out of range ‚Äã

  • Severity: ERROR
  • Fix: Correct selector width or bounds; ensure compile-time indices fall within valid range.

Assignments & Exclusive Assignment Rule ‚Äã

ASSIGNMENTS_AND_EXCLUSIVE.ASSIGN_WIDTH_NO_MODIFIER ‚Äî Width mismatch without z/s ‚Äã

  • Severity: ERROR
  • Cause: =, =>, <= used when widths differ (modifier required).
  • Fix: Use =z/=s, =>z/=>s, or resize operands so widths match.

ASSIGNMENTS_AND_EXCLUSIVE.ASSIGN_TRUNCATES ‚Äî Assignment would truncate RHS ‚Äã

  • Severity: ERROR
  • Fix: Prevent truncation by slicing explicitly or making LHS wide enough; avoid implicit truncation.

ASSIGNMENTS_AND_EXCLUSIVE.ASSIGN_SLICE_WIDTH_MISMATCH ‚Äî Slice width mismatch ‚Äã

  • Severity: ERROR
  • Fix: Ensure source expression width equals the slice width.

ASSIGNMENTS_AND_EXCLUSIVE.ASSIGN_CONCAT_WIDTH_MISMATCH ‚Äî Concatenation width mismatch ‚Äã

  • Severity: ERROR
  • Fix: Make expression width equal sum of LHS part widths (or use extension modifiers if allowed).

ASSIGNMENTS_AND_EXCLUSIVE.ASSIGN_MULTIPLE_SAME_BITS ‚Äî Exclusive assignment violation ‚Äã

  • Severity: ERROR
  • Cause: More than one assignment to same bits along some execution path.
  • Fix: Rework control flow so assignments are in mutually exclusive branches or merge into a single assignment using ternary/select.

ASSIGNMENTS_AND_EXCLUSIVE.ASSIGN_INDEPENDENT_IF_SELECT ‚Äî Independent chains conflict ‚Äã

  • Severity: ERROR
  • Cause: Separate IF chains at same nesting level each assign same identifier (compiler treats as concurrent).
  • Fix: Combine into single SELECT/IF/ELIF chain or restructure to make assignments exclusive.

ASSIGNMENTS_AND_EXCLUSIVE.ASSIGN_SHADOWING ‚Äî Shadowing assignment (root + nested) ‚Äã

  • Severity: ERROR
  • Fix: Avoid assigning at root and again in nested blocks to same bits; use conditional assignments consistently.

ASSIGNMENTS_AND_EXCLUSIVE.ASSIGN_SLICE_OVERLAP ‚Äî Overlapping part-selects assigned in same path ‚Äã

  • Severity: ERROR
  • Fix: Ensure part-selects are non-overlapping on any execution path or combine into single concatenation assignment.

ASSIGNMENTS_AND_EXCLUSIVE.ASYNC_UNDEFINED_PATH_NO_DRIVER ‚Äî ASYNC leaves net without driver ‚Äã

  • Severity: ERROR
  • Cause: Conditional ASYNCHRONOUS assignment path leaves a wire/port undriven (no ELSE) and no other driver exists.
  • Fix: Provide DEFAULT branch or alternate driver; ensure every execution path that reads the net has a driver.

ASYNCHRONOUS Block Rules ‚Äã

ASYNC_BLOCK_RULES.ASYNC_INVALID_STATEMENT_TARGET ‚Äî LHS not assignable ‚Äã

  • Severity: ERROR
  • Fix: LHS must be assignable (wire, port, register current-value read in ASYNC can't be written there).

ASYNC_BLOCK_RULES.ASYNC_ASSIGN_REGISTER ‚Äî Register written in ASYNCHRONOUS ‚Äã

  • Severity: ERROR
  • Fix: Move register writes into SYNCHRONOUS blocks.

ASYNC_BLOCK_RULES.ASYNC_ALIAS_LITERAL_RHS ‚Äî Alias with literal RHS in ASYNC ‚Äã

  • Severity: ERROR
  • Cause: Using a = 1'b1; in ASYNCHRONOUS is forbidden (alias RHS must not be bare literal).
  • Fix: Use directional drive a <= 1'b1; or a => 1'b1; instead.

ASYNC_BLOCK_RULES.ASYNC_FLOATING_Z_READ ‚Äî All drivers z on read ‚Äã

  • Severity: ERROR
  • Fix: Ensure at least one driver supplies 0/1 for read conditions or avoid reading when bus released.

SYNCHRONOUS Block Rules ‚Äã

SYNC_BLOCK_RULES.SYNC_MULTI_ASSIGN_SAME_REG_BITS ‚Äî Multiple reg assignments in path ‚Äã

  • Severity: ERROR
  • Fix: Ensure each register bit assigned at most once per path; move assignments into mutually exclusive branches or combine.

SYNC_BLOCK_RULES.SYNC_ROOT_AND_CONDITIONAL_ASSIGN ‚Äî Root + conditional conflict ‚Äã

  • Severity: ERROR
  • Fix: Do not assign same register at root and inside nested condition; place all domain assignments within consistent conditional structure.

SYNC_BLOCK_RULES.SYNC_SLICE_WIDTH_MISMATCH ‚Äî Register slice width mismatch ‚Äã

  • Severity: ERROR
  • Fix: Ensure RHS width equals slice width.

SYNC_BLOCK_RULES.SYNC_CONCAT_DUP_REG ‚Äî Concatenation contains same register multiple times ‚Äã

  • Severity: ERROR
  • Fix: Use unique registers in concatenation LHS.

SYNC_BLOCK_RULES.SYNC_NO_ALIAS ‚Äî Aliasing not allowed in synchronous blocks ‚Äã

  • Severity: ERROR
  • Fix: Use directional operators (<=, =>) only; alias operators are forbidden in SYNCHRONOUS.

SYNC_BLOCK_RULES.DOMAIN_CONFLICT / MULTI_CLK_ASSIGN / DUPLICATE_BLOCK ‚Äî Clock/CDC domain errors ‚Äã

  • Severity: ERROR
  • Cause: Register read/write used in block whose CLK doesn't match home domain; same register assigned in different clock blocks; duplicate synchronous block for a clock.
  • Fix: Respect Register locality/home-domain rules; use CDC entries to cross domains; consolidate per-clock logic into single SYNCHRONOUS block.

SYNC_BLOCK_RULES.CDC_SOURCE_NOT_REGISTER / CDC_BIT_WIDTH_NOT_1 ‚Äã

  • Severity: ERROR
  • Fix: CDC sources must be REGISTER; BIT type registers must have width 1.

SYNC_BLOCK_RULES.SYNC_CLK_WIDTH_NOT_1 ‚Äî Clock signal not width-1 ‚Äã

  • Severity: ERROR
  • Fix: SYNCHRONOUS CLK signal must have width [1].

SYNC_BLOCK_RULES.SYNC_MISSING_CLK ‚Äî Missing CLK parameter ‚Äã

  • Severity: ERROR
  • Fix: SYNCHRONOUS block must declare a CLK parameter.

SYNC_BLOCK_RULES.SYNC_EDGE_INVALID ‚Äî Invalid edge value ‚Äã

  • Severity: ERROR
  • Fix: EDGE must be Rising, Falling, or Both.

SYNC_BLOCK_RULES.SYNC_RESET_ACTIVE_INVALID ‚Äî Invalid RESET_ACTIVE ‚Äã

  • Severity: ERROR
  • Fix: RESET_ACTIVE must be High or Low.

SYNC_BLOCK_RULES.SYNC_RESET_TYPE_INVALID ‚Äî Invalid RESET_TYPE ‚Äã

  • Severity: ERROR
  • Fix: RESET_TYPE must be Clocked or Immediate.

SYNC_BLOCK_RULES.SYNC_RESET_WIDTH_NOT_1 ‚Äî Reset signal not width-1 ‚Äã

  • Severity: ERROR
  • Fix: RESET signal must have width [1].

SYNC_BLOCK_RULES.SYNC_UNKNOWN_PARAM ‚Äî Unknown SYNCHRONOUS parameter ‚Äã

  • Severity: ERROR
  • Fix: Valid parameters are CLK, RESET, EDGE, RESET_ACTIVE, RESET_TYPE.

  • Severity: ERROR
  • Cause: A symbolic link inside the sandbox resolves to a target outside the sandbox.
  • Fix: Remove the symlink or add the target directory to --sandbox-root.
